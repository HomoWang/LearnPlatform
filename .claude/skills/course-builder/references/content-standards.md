# 內容品質標準（每種區塊的寫法與好壞對照）

總原則：讀者是**完全的小白**，且**沒有老師可以問**。每一段內容要通過這個測試：
「一個不懂的人照著做，會不會在某一步卡住、而頁面上找不到答案？」會 → 內容不合格。

## teachingSteps（學習步驟，5–6 步）

每一步必須是「可執行的動作」或「可驗證的判斷」，不是主題摘要。

- ❌ 大綱式：「理解 DB2 名詞地圖：Instance、Database、Tablespace」（這是目錄，照著做不出任何東西）
- ✅ 教學式：「開兩個資料庫連線做實驗：A 更新不提交、B 讀同一筆，觀察兩種隔離層級下 B 讀到什麼」

自查法：把每一步唸出來問「小白現在要動手做什麼？」答不出來就重寫。

## example（範例）

必須是以下三者之一，禁止「文字描述的清單」冒充範例：
1. 可直接複製執行的指令/程式碼/設定檔（附一兩行說明重點）
2. 真實長相的輸出/Log ＋ 怎麼讀它（沒有設備的主題——F5、防火牆——用這種）
3. 具體情境的數字推演（IP: 192.168.10.25 / Mask: /24 → Network: 192.168.10.0…）

寫具體工具指令前先查證時效（鐵律 6）。

## practiceTasks（練習任務，3–5 個）

每個任務要有**看得到的交付物**：一份表、一張圖、一段可執行的 SQL、一份紀錄。
「列出」「整理」型任務最多佔三分之一，且要指明交付格式。

- ❌ 「理解 NAT 的運作」
- ✅ 「畫出一個內部 IP 經 NAT 出網際網路的流程，標出轉換前後的 IP」

## beginnerChecklist（小白檢核，4–5 條）

一律「我知道～」「我能～」開頭的自評句，每條對應本章一個具體知識點，
且答案必須能在本章內容裡找到（不能檢核沒教過的東西）。

## handson（手把手解答）— 本平台的靈魂區塊

定位：使用者先自己做練習任務，卡住時展開它。所以它是**練習任務的逐題解答＋帶做流程**。
一個長字串、`\n` 換行、用【第 N 步】或【任務 N 解答】分段。必備元素：

1. **從零的起點**：第一步從「開始鍵 → 輸入 powershell」這種粒度開始，不假設任何前置操作。
2. **指令原文**：可直接複製，行首用 `>` 標示。
3. **預期輸出**：成功長什麼樣（貼輸出關鍵行），讓使用者能對答案。
4. **失敗長什麼樣＋代表什麼**：「要求等候逾時 → 表示連路由器都到不了，問題在線路」。
5. **每步的為什麼**：不只給指令，講這步在驗證什麼。
6. **練習任務逐題解答**：用【任務 N 解答】標明，計算題給完整推導、設計題給範本。
7. **常見卡點收尾**：2–3 個安裝/環境層級的坑（Port 被占用、WSL2 沒開）與解法。
8. 指令參數逐一解釋（`-p 5432:5432` 是什麼意思），比喻優先（Docker＝軟體即食包）。

長度基準：一章的 handson 約 30–60 行文字。太短通常代表跳步了。

## commonIssues / troubleshooting / interviewPoints

- commonIssues 寫**真實工單會出現的現象**（「使用者換電腦後 Port 被鎖住」），不是抽象風險。
- troubleshooting 要有**順序**（先收集什麼 → 再查什麼 → 最後看什麼），不是無序清單。
- interviewPoints 用「說明～」「描述～」句型，對應真實面試問法。
- **鐵律 3 自查**：這三個欄位出現的每個概念，必須在本課或前面的課教過。

## glossary（名詞字典條目）

- 1–2 句白話，第一句講「它是什麼／像什麼」，第二句講「維運現場它出現在哪／壞了長什麼樣」。
- 能比喻就比喻：Gateway＝出門的第一站、Index＝書的目錄、Image vs Container＝食譜 vs 做出來的菜。
- aliases 放同義寫法（縮寫、英文全名、中文名），供自動比對命中。
- 一組總是一起出現的概念可合併一條（用 aliases 涵蓋），避免字典碎片化。

## labs

- steps 是骨架（一句話一步）；**stepGuides 按索引對齊**，每步給具體指令/設定與原因，粒度同 handson。
- prerequisites 引用的能力必須是前面課程教過的。
- portfolioTemplate：第一人稱、一句話涵蓋「做了什麼＋用了什麼＋能證明什麼能力」，可直接貼履歷。

## interviewQuestions

- referenceAnswer 2–4 句，結構優先：「差異對比」「排查順序」「原因 → 處理 → 長期改善」。
- troubleshooting 型答案一定包含排查順序；scenario 型包含通報/紀錄等流程元素。
- 每題 relatedCourses / relatedLabs 必填，指向真正教過這題答案的課。

## 語言

繁體中文（台灣用語：資料庫不是数据库、網路不是网络、伺服器不是服务器），
技術名詞保留英文，中英之間的行文自然即可。語氣：像坐在旁邊帶你做的學長，不是教科書。
