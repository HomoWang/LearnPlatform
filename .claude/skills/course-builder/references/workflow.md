# 從方向/JD 到完整路線的產出流程

適用：使用者說「我想學 X」「幫我把這幾份 JD 變成課程」——他可能對 X 一無所知，
知識點推導是你的責任。整個流程的核心思想：**JD 是冰山一角，你要把水面下的地基全部挖出來**。

## 第 1 步：需求訪談（缺什麼就問，但別問使用者答不出的）

可以問：目標職缺 JD 原文（越多份越好）、預計投入時間、家用電腦規格/作業系統、已會的東西（可能是「都不會」）。
**不要問**：「你想學哪些知識點？」「需要涵蓋哪些主題？」——跨領域的使用者答不出來，這是你要產出的東西。

## 第 2 步：JD → 知識點地圖（本 skill 最重要的一步）

### 2a. 抽詞
把每份 JD 逐句拆，抽出所有：工具/產品名（F5、GitLab、Oracle）、技術名詞（VLAN、CI/CD）、
職責動詞（監控、排查、備份、上版、稽核）。多份 JD 取聯集，並記錄每個詞出現頻率（頻率高＝核心）。

### 2b. 地基遞迴推導（讓跨領域從 0 開始成為可能）
對每個 JD 名詞反覆問：「**這個詞背後假設你已經會什麼？**」一路遞迴到小白起點。範例：

```
JD 寫「F5 維運」
 └─ 假設你懂 Load Balancer 分流 → 假設你懂 HTTPS/憑證
     └─ 假設你懂 HTTP 狀態碼、TLS → 假設你懂 TCP、Port
         └─ 假設你懂 IP/Subnet → 假設你會用終端機打指令 ← 小白起點，遞迴停止
```

推導出來但 JD 沒寫的詞（TCP、憑證鏈、終端機操作）就是**地基知識點**——
JD 從來不寫地基，因為它假設應徵者已經會；但小白課程的價值恰恰在地基。
如果你自己對該領域不熟，先 WebSearch 該職缺的典型技能樹與入門教材目錄，再做推導。

### 2c. 分層
把全部知識點分四層：
- **環境準備**：免費、家用電腦能建的練習環境（模擬器、Docker、免費雲方案）＋該領域的基本工具操作。
- **地基**：2b 推導出來、JD 沒寫但不會就學不下去的觀念。
- **核心**：JD 明寫、出現頻率高的技能。深度要到「能實作＋能排查＋能面試說明」。
- **進階/認知**：JD 偶爾出現或寫「加分」的項目。深度到「能講清楚它解決什麼問題」即可（如 K8s 認知課），明說不必會操作。

### 2d. 排階段
排序原則：環境 → 地基 → 核心（按依賴順序）→ 進階 → 整合（監控、合規、事件處理、面試準備類的收尾內容）。
每個階段 = 一門課（大主題可拆兩門）；每門課 3–5 個章節為宜，超過就拆課。
**每一個階段都必須有課，不做「先佔位、之後再補」的空階段。**

### 2e. 給使用者確認
輸出一張表：階段 → 課程標題 → 一句話說明 → 章節清單 → 對應的 JD 原句。
「對應的 JD 原句」讓不懂技術的使用者也能驗證「這些課確實在回應我要的職缺」。
確認通過才進第 3 步。

## 第 3 步：產出順序（依賴方向由上游到下游）

1. learningPaths（stages 定稿）
2. courses + lessons 骨架（objective/content/commonIssues/troubleshooting/interviewPoints）
3. lessonTeachings（每章教學；入門段含 handson）
4. labs（每門核心課至少一個；含 stepGuides 與 portfolioTemplate）
5. skills（每課 1–2 個「我能～」）
6. interviewQuestions（每路線 15 題起、四題型平衡、relatedCourses/relatedLabs 必填）
7. jobMappings（每條 JD 職責一筆）
8. glossary（掃一遍新內容，把所有新名詞補進字典）

寫 commonIssues 時同步自查鐵律 3：**這條 issue 裡的概念有沒有課文教過？**沒有就回頭補章節或補教學段落。

## 第 4 步：驗證

```
node .claude/skills/course-builder/scripts/validate.mjs
npm run build
```
validate 會列出「尚無 handson 的章節」清單——用它管理內容債，回報時如實告知覆蓋範圍。

## 增量任務的流程（補內容、加章節）

- 「補手把手」：從 validate 輸出的無 handson 清單中，**按學習順序**（路線 order → 課內順序）取下一批，通常一批 10–16 章。
- 「加一門課/章節」：照第 3 步的順序走該課的 2→8，不要只加 courses 骨架。
- 改既有內容：不改 ID；牽動的 glossary、面試題、Lab 引用一併檢查。

## 審查任務

使用者說「審查/review 課程」時改走 [review-checklist.md](review-checklist.md)，
只評估、列問題與優先序，**未經指示不動手改**。
