# 十七、請注意的限制

因為此平台要部署在 GitHub Pages 並使用 localStorage：

1. 不要設計後端登入
2. 不要設計多人資料庫
3. 不要把資料存在伺服器
4. 不要存放機密資訊
5. 清除瀏覽器資料會導致進度消失
6. 請務必提供 JSON 匯出 / 匯入備份
7. 換電腦時需透過匯入 JSON 還原資料

# 十九、整體開發目標總結

請幫我建立一個可部署到 GitHub Pages 的單人 IT 學習管理平台。

這個平台要支援三條學習路線：

1. 網路工程師路線
   包含 Firewall、VPN、ACL、Proxy、Reverse Proxy、WAF、F5、Load Balancer、企業網路部署、VDI、PRTG、資安合規與異常排除。

2. DBS / DBA 路線
   包含 SQL、資料結構設定、Schema 設計、帳號權限切分、DB Migration、備份還原、DB2、Oracle、資料庫監控、效能調教、虛擬化平台與資料庫安全合規。

3. DevOps / DevSecOps / AI Platform 路線
   包含 GitLab 管控、GitLab 維運、CI/CD Pipeline、SonarQube、OWASP ZAP、NuGet / pip 套件弱點檢查、Docker、IaC、自動化部署與企業 AI 平台落地。

平台不是單純顯示課程，而是要能做到：

1. 學習規劃
2. 進度追蹤
3. 實作 Lab 管理
4. 能力檢核
5. 筆記整理
6. 面試題練習
7. 職缺能力對應
8. 作品集整理
9. JSON 備份還原

請用乾淨、專業、可維護的方式開發，並讓後續可以很容易新增課程、Lab、能力項目與面試題。
